# おすすめ商品セクション デザイン仕様書

## 概要

bier.jpトップページの「おすすめ商品」セクションのデザイン改善プロジェクトにおける、3つのデザイン案の詳細仕様をまとめたドキュメントです。

---

## デザイン案一覧

### デザインA: シンプルグリッド

#### コンセプト
商品をカード形式のグリッドで表示する、モダンで標準的なレイアウトです。多くのECサイトで採用されている定番のデザインパターンを踏襲しています。

#### 特徴
- **レイアウト**: 3カラムグリッド（レスポンシブ対応）
- **カードデザイン**: 角丸（border-radius: 1rem）、シャドウ付き
- **商品画像**: カード上部に配置、高さ192px固定
- **情報表示**: 商品名、銘柄、在庫数、アルコール度数、説明文

#### カラースキーム
| 要素 | カラー |
|------|--------|
| 背景 | グラデーション（gray-100 → white） |
| カード背景 | 白（#FFFFFF） |
| 商品名 | イエロー系（text-yellow-600） |
| テキスト | グレー系（text-gray-500, text-gray-600） |

#### メリット
- シンプルで直感的なUI
- 商品の一覧性が高い
- 実装が容易

#### デメリット
- 多くの情報を表示すると煩雑になりやすい
- 他サイトとの差別化が難しい

---

### デザインB: ホバー詳細表示

#### コンセプト
カードにカーソルを合わせると、詳細情報がオーバーレイで表示されるインタラクティブなレイアウトです。ユーザーの関心を引く動的なデザインが特徴です。

#### 特徴
- **レイアウト**: 3カラムグリッド（レスポンシブ対応）
- **カードデザイン**: 角丸（border-radius: 1rem）、シャドウ付き、ホバー時拡大アニメーション
- **商品画像**: カード上部に配置、高さ224px固定
- **インタラクション**: ホバー時にオーバーレイ表示（不透明度95%）

#### ホバーエフェクト
```css
transform: scale(1.05);
transition: transform 0.3s;
```

#### オーバーレイ表示項目
- アルコール度数
- 原産国
- 容量
- 醸造所
- 特徴（フレーバーノート）

#### カラースキーム
| 要素 | カラー |
|------|--------|
| 背景 | ライトグレー（bg-gray-50） |
| カード背景 | 白（#FFFFFF） |
| カテゴリバッジ | イエロー系（bg-yellow-100, text-yellow-800） |
| CTAボタン | イエロー系（bg-yellow-500） |

#### メリット
- 情報の階層化により初期表示がすっきり
- インタラクティブでユーザーの興味を引く
- 詳細情報へのアクセスが容易

#### デメリット
- タッチデバイスではホバーが使えない
- JavaScriptに依存する場合がある

---

### デザインC: 横長カード

#### コンセプト
画像を左に、情報を右に配置した横長のカードレイアウトです。一覧性が高く、情報量が多い場合に適しています。

#### 特徴
- **レイアウト**: 2カラムグリッド（レスポンシブ対応）
- **カードデザイン**: 横長フレックスボックス、角丸（border-radius: 1rem）
- **商品画像**: 左側に配置、幅192px固定
- **情報配置**: 右側に商品名、カテゴリ、価格、在庫、説明文を縦に配置

#### ホバーエフェクト
```css
box-shadow: 変化（shadow-xl → shadow-2xl）
transition: box-shadow 0.3s;
```

#### カラースキーム
| 要素 | カラー |
|------|--------|
| 背景 | ライトグレー（bg-gray-50） |
| カード背景 | 白（#FFFFFF） |
| 商品名 | ダークグレー（text-gray-800） |
| CTAボタン | イエロー系（bg-yellow-500） |

#### メリット
- 情報量が多くても見やすい
- 商品説明を十分に表示できる
- スキャンしやすいレイアウト

#### デメリット
- 画面上の商品表示数が少なくなる
- 商品画像のインパクトが弱まる

## 推奨デザイン案

### 第1推奨: デザインB（ホバー詳細表示）

#### 推奨理由

1. **ユーザーエンゲージメント向上**
   - インタラクティブな要素がユーザーの興味を引く
   - 詳細情報への自然な導線を提供

2. **情報の階層化**
   - 初期表示はシンプルに保ちつつ、詳細情報も容易にアクセス可能
   - ドイツビールの特徴（原産国、醸造所、フレーバー）を効果的に伝達

3. **モダンなUI/UX**
   - 競合サイトとの差別化が可能
   - ブランドイメージの向上

#### 実装時の考慮事項

- タッチデバイス対応として、タップで詳細表示に切り替える実装を追加
- パフォーマンス最適化のため、画像の遅延読み込みを検討
- アクセシビリティ対応として、キーボード操作でも詳細表示可能にする

### 第2推奨: デザインC（横長カード）

商品説明を重視する場合や、情報量が多い商品を扱う場合に推奨されます。スマートフォンでの閲覧が多い場合は、縦長デザインへの自動切り替えも考慮してください。
